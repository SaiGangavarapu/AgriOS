# What Changed — Complete Database Design v1.0

This release adds the complete initial AgriOS database-design baseline.

It includes:

1. Database principles and naming conventions
2. PostgreSQL and PostGIS schema ownership
3. Tenant and programme partitioning
4. Logical and physical table design for all major bounded contexts
5. Geospatial and boundary-versioning design
6. Time-series telemetry and weather storage
7. Object-storage metadata and retention
8. Search and reporting projection strategy
9. Indexing, partitioning, archival, and retention
10. Flyway migration governance
11. Seed and reference-data strategy
12. Audit, consent, privacy, and data-protection design
13. Backup, restore, reconciliation, and integrity controls
14. Database-to-SRS and database-to-context traceability

No application implementation code is included.
