# Research Validation Backlog

## 1. Purpose

This backlog identifies subjects that require crop-specific, region-specific, legal, or expert validation before they become executable recommendations.

## 2. Priority A — before MVP agronomy recommendations

- soil-test interpretation rules by state and laboratory method
- supported crop catalogue for pilot districts
- variety catalogue and official release status
- crop-stage definitions
- sowing windows
- nutrient recommendation sources
- irrigation thresholds
- restricted and prohibited input rules
- expert-review workflow
- weather-source usage rights and API availability

## 3. Priority B — before pilot

- district and village pilot selection
- local soil laboratory workflow
- KVK or agricultural-university collaboration
- representative farmer personas
- local-language terminology
- connectivity survey
- IoT sensor shortlist
- calibration protocol
- field validation methodology
- consent and farmer-data agreement

## 4. Priority C — before automation

- pump-controller electrical safety
- fail-safe design
- water-flow verification
- sensor reliability benchmarks
- actuator authorization
- offline conflict handling
- emergency manual override
- liability and support process

## 5. Priority D — before machine learning

- minimum dataset size
- ground-truth definition
- missing-data policy
- bias and geographic-transfer testing
- model confidence calibration
- agronomist evaluation
- monitoring and rollback
- farmer-impact study

## 6. Initial pilot recommendation

Select:

- one state
- two or three districts
- three to five crops
- multiple irrigation conditions
- 50 to 100 farms
- 10 to 20 IoT-enabled farms
- one agricultural expert institution
- one soil laboratory

## 7. Open architectural questions

- shared ERP foundation integration strategy
- standalone versus shared identity
- tenancy model
- farmer-owned versus institution-managed devices
- weather licensing
- satellite-provider strategy
- consent interoperability
- marketplace integration boundary
