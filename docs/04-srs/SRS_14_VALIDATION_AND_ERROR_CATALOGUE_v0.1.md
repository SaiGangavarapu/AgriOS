# SRS 14 — Validation and Error Catalogue

## Error structure

Every API error should include:

- code
- message key
- localized message
- field where applicable
- correlation id
- retryable flag
- details safe for the actor

## Farmer errors

- `FARMER_DUPLICATE_CONFIRMED`
- `FARMER_DUPLICATE_REVIEW_REQUIRED`
- `FARMER_PROFILE_SUSPENDED`
- `FARMER_VERSION_CONFLICT`
- `FARMER_VERIFICATION_EVIDENCE_INVALID`

## Farm errors

- `FARM_PRIMARY_OPERATOR_REQUIRED`
- `FARM_ACTIVE_CROP_CYCLE_EXISTS`
- `FARM_ACCESS_DENIED`

## Field errors

- `FIELD_GEOMETRY_INVALID`
- `FIELD_AREA_IMPLAUSIBLE`
- `FIELD_OVERLAP_REVIEW_REQUIRED`
- `FIELD_OUTSIDE_SUPPORTED_GEOGRAPHY`
- `FIELD_ACTIVE_BOUNDARY_REQUIRED`
- `FIELD_VERSION_CONFLICT`

## Tenure errors

- `TENURE_CONFLICT_REVIEW_REQUIRED`
- `TENURE_EXPIRED`
- `TENURE_EVIDENCE_REQUIRED`
- `TENURE_ACCESS_DENIED`

## Access errors

- `ACCESS_SCOPE_INVALID`
- `ACCESS_DELEGATION_EXPIRED`
- `ACCESS_REAUTHENTICATION_REQUIRED`
- `ACCESS_PERMISSION_DENIED`

## Consent errors

- `CONSENT_REQUIRED`
- `CONSENT_EXPIRED`
- `CONSENT_REVOKED`
- `CONSENT_PURPOSE_MISMATCH`
- `CONSENT_RECIPIENT_MISMATCH`
- `CONSENT_SCOPE_EXCEEDED`

## Technical errors

- `REQUEST_IDEMPOTENCY_CONFLICT`
- `RESOURCE_VERSION_CONFLICT`
- `DEPENDENCY_TEMPORARILY_UNAVAILABLE`
- `INTERNAL_PROCESSING_ERROR`
