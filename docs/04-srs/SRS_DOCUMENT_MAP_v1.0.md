# SRS Document Map v1.0

## Business source

- BRD Parts 1–6
- Requirements BR-001 through BR-340

## Domain source

- Domain Architecture Parts 1–3
- ADR-0001 through ADR-0008

## Downstream artifacts

The SRS is the source for:

- solution architecture
- database design
- API specification
- UI/UX specification
- implementation backlog
- test strategy
- release acceptance

## Requirement format

Each functional requirement uses `FR-<DOMAIN>-NNN`.

Related categories:

- `VR-` validation
- `SR-` security
- `DR-` data
- `IR-` integration
- `NFR-` non-functional
- `AC-` acceptance criterion
