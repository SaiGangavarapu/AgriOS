# AgriOS Architecture Principles

1. Field-specific decisions over generic advice.
2. Official and peer-reviewed sources over unverified commercial claims.
3. Deterministic safety rules before AI-generated language.
4. Human expert escalation for low-confidence or high-risk cases.
5. Offline-first farmer experience.
6. Modular monolith before premature microservices.
7. Event-driven integration where data changes matter.
8. Complete auditability for recommendations and farmer actions.
9. Consent-based sharing of farmer data.
10. Clear separation between agronomic advice and commercial promotion.
11. Progressive IoT adoption based on demonstrated value.
12. Laboratory soil tests remain authoritative for nutrient planning.
