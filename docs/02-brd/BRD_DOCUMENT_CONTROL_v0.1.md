## Change-control rule

Material changes to business scope, target users, core outcomes, safety boundaries, or regulatory assumptions must be recorded in Git history and, where architectural impact exists, in an ADR.

## Traceability rule

Each business requirement must have:

- unique identifier
- title
- source
- rationale
- priority
- acceptance measure
- dependent capability
- downstream SRS reference
- implementation status
